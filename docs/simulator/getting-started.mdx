# Getting Started

## System Requirements

Before installing, ensure you have:

- **Python ≥ 3.11** ([download](https://www.python.org/downloads/))
- **Node.js ≥ 18** ([download](https://nodejs.org/))
- **`uv` package manager** (install via `pip install uv` or [see docs](https://docs.astral.sh/uv/getting-started/installation/))
- **Git** (to clone the repository)

Verify installation:

```bash
python --version      # Should be 3.11+
node --version        # Should be 18+
npm --version         # Should be 9+
uv --version          # Should be 0.1.0+
```

---

## Clone and Navigate

```bash
git clone https://github.com/your-org/ocean-sciences.git
cd ocean-sciences
```

(Replace with your actual repository URL.)

---

## Backend Setup

The backend is a **FastAPI** server that runs simulations. Set it up in a separate terminal:

### 1. Create a Virtual Environment

```bash
cd backend
uv sync
```

This command:
- Creates a `.venv` virtual environment
- Installs all dependencies from `pyproject.toml`
- Takes ~30 seconds on first run

### 2. Start the Server

```bash
uv run uvicorn api:app --reload
```

**Expected output**:
```
INFO:     Uvicorn running on http://0.0.0.0:8000
INFO:     Application startup complete
```

The `--reload` flag enables hot-reloading during development; remove it for production.

### 3. Verify the Server

In a third terminal, test connectivity:

```bash
curl http://localhost:8000/api/health
```

**Expected response**:
```json
{"status": "ok"}
```

---

## Frontend Setup

The frontend is a **React + Vite** web app. Set it up in a separate terminal:

### 1. Install Dependencies

```bash
cd frontend
npm install
```

This installs ~163 packages listed in `package.json`. Takes ~1–2 minutes on first run.

### 2. Start the Development Server

```bash
npm run dev
```

**Expected output**:
```
VITE v5.4.21 ready in 205ms

➜  Local:   http://localhost:5173/
➜  press h + enter to show help
```

The Vite dev server automatically reloads when you edit files.

### 3. Open in Browser

Navigate to **http://localhost:5173** in your web browser.

### Click "Run Simulation"

1. Leave all parameters at defaults (the app auto-runs once on page load anyway)
2. Click the green **"Run Simulation"** button
3. Watch the visualizations populate:
   - **Top-left**: Two-box ocean diagram with an arrow showing flow direction/strength
   - **Top-right**: Comparison of model's q_sv vs. observed RAPID-MOCHA mean
   - **Middle**: Time series of q, S₁, S₂ vs. time (years)
   - **Bottom-left**: Phase space portrait (ΔS vs. q)
   - **Bottom-right**: [empty until you click "Compute Bifurcation Diagram"]

The simulation took ~1–2 seconds. The system quickly equilibrated to a steady state (note the flat time series near t=3000 years).

## Understanding the Layout

### Left Panel: Controls

- **Preset buttons** (top): "Normal AMOC", "Weakened", "Collapsed", "Stommel 1961"
- **Temperature & Salinity sliders**: T₁, T₂, S₁ᵉᵍ, S₂ᵉᵍ, S₁ᵢⁿⁱᵗ, S₂ᵢⁿⁱᵗ
- **Physics Parameters sliders**: α, β, k, λ — k and λ are on **logarithmic** sliders
- **Stochastic Forcing**: noise_amplitude
- **Integration**: t_max, dt
- **Run Simulation** (green): Execute one simulation with current parameters
- **Reset** (red): Restore all sliders to Stommel 1961 defaults
- **Compute Bifurcation Diagram** (blue): Sweep T₂ to generate the S-curve

> There is no "Export" button and no freshwater forcing (F) slider in this version. The bifurcation parameter is **ΔT = T₁ − T₂**, controlled by the T₂ slider. To get raw data, hit the API directly (see [API Reference](/api-reference/overview)).

### Right Panel: Visualizations (6 panels)

See [Visualizations](/simulator/visualizations) for detailed explanation of each.

## Smoke Tests: Verify Everything Works

Run these commands to confirm the full stack is operational:

### Backend Health

```bash
curl -s http://localhost:8000/api/health | jq .
# Expected: {"status": "ok"}
```

### API Simulation Endpoint

```bash
curl -s -X POST http://localhost:8000/api/simulate \
  -H "Content-Type: application/json" \
  -d '{
    "T_1": 25,
    "T_2": 5,
    "S_1_eq": 36,
    "S_2_eq": 34,
    "S_1_init": 36,
    "S_2_init": 34,
    "alpha": 2.0e-4,
    "beta": 1.0e-3,
    "k": 3.0e-9,
    "lam": 3.0e-11,
    "t_max": 3000,
    "dt": 1,
    "noise_amplitude": 0
  }' | jq '.q_sv[-1]'
# Expected: ~15 (final circulation in Sv)
```

### API Presets Endpoint

```bash
curl -s http://localhost:8000/api/presets | jq '.presets[].name'
# Expected:
# "normal_amoc"
# "weakened_amoc"
# "collapsed_amoc"
# "stommel_original"
```

### Frontend Page Load

```bash
curl -s http://localhost:5173 | grep -i "<title>" | head -1
# Expected: <title>Stommel THC Simulator</title>
```

## Troubleshooting

### Issue: "Port 8000 is already in use"

**Solution**: Either kill the process on port 8000 or use a different port:

```bash
# Find and kill process on port 8000 (macOS/Linux)
lsof -ti:8000 | xargs kill -9

# Or use a different port
uv run uvicorn api:app --reload --port 8001
```

### Issue: "uv: command not found"

**Solution**: Install uv globally:

```bash
pip install uv
uv --version
```

Or use the installation script:

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

### Issue: "npm install fails" or takes forever

**Solution**: Clear npm cache and try again:

```bash
npm cache clean --force
rm -rf node_modules package-lock.json
npm install
```

### Issue: "Python version mismatch" (need 3.11+)

**Solution**: Check your Python version and install a newer one if needed:

```bash
python --version
# If < 3.11, install Python 3.11 or 3.12
```

On macOS with Homebrew:
```bash
brew install python@3.12
brew link python@3.12
```

### Issue: Browser shows blank page / "Cannot GET /"

**Solution**: Make sure the frontend dev server is running and on port 5173:

```bash
cd frontend
npm run dev
# Verify output shows "Local: http://localhost:5173/"
```

Then clear browser cache (Ctrl+Shift+R or Cmd+Shift+R) and reload.

### Issue: Simulations show "Failed to fetch" errors

**Solution**: Ensure backend is running on port 8000:

```bash
# Terminal 1
cd backend
uv run uvicorn api:app --reload

# Check connectivity
curl http://localhost:8000/api/health
```

The frontend expects the backend at `http://localhost:8000` (the Vite dev server proxies `/api/*` requests to it). If using a different port, update the Vite config or backend server URL.
