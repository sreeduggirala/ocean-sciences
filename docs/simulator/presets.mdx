# Preset Scenarios

The simulator includes **four preset parameter sets** representing distinct regimes of the Atlantic Meridional Overturning Circulation. Click any of the green preset buttons at the top of the control panel to instantly load them.

All four presets share the same **Stommel 1961 physical coefficients** — they differ primarily in **T₂** (the polar temperature), which sets the temperature gradient ΔT = T₁ − T₂ that drives circulation. This is the bifurcation parameter in the relaxation model.

| Coefficient | All presets |
|-------------|-------------|
| α (thermal expansion) | 2 × 10⁻⁴ /°C |
| β (haline contraction) | 1 × 10⁻³ /psu |
| k (circulation) | 3 × 10⁻⁹ 1/s |
| λ (relaxation rate) | 3 × 10⁻¹¹ 1/s (~1000-yr timescale) |
| t_max | 3000 years |
| dt | 1 year |
| noise | 0 |

---

## Preset 1: Normal AMOC

**Backend name**: `normal_amoc`
**Description**: Stable thermally-driven AMOC. The original Stommel (1961) baseline.

### Parameters (only what differs from the shared coefficients above)

| Parameter | Value |
|-----------|-------|
| T₁ | 25 °C |
| T₂ | 5 °C |
| **ΔT** | **20 °C** |
| S₁ᵉᵍ | 36 psu |
| S₂ᵉᵍ | 34 psu |
| S₁ᵢⁿⁱᵗ | 36 psu |
| S₂ᵢⁿⁱᵗ | 34 psu |

### What to Observe

**Time series**:
- Quick equilibration to a strong steady state
- Final q_sv typically ~15 Sv (matches RAPID-MOCHA observations)
- Salinity difference ΔS settles at ~2 psu

**Phase portrait**:
- Trajectory from (S₁ᵢⁿⁱᵗ, S₂ᵢⁿⁱᵗ) = (36, 34) drifts a little, then settles on the upper-branch attractor
- Red star and green circle both on upper branch

**Two-box diagram**:
- Thick northward arrow (large positive q)

**AMOC comparison**:
- Blue bar matches the RAPID-MOCHA grey bar (~16.9 Sv)

**Bifurcation diagram**:
- Orange X marker sits at ΔT = 20 °C, well to the right of the collapse threshold
- Lots of "safety margin"

### Physical Interpretation

This represents the **modern Atlantic AMOC**: strong thermal forcing wins over haline freshening; circulation is self-sustaining and resilient.

### Experiments to Try

1. **Verify robustness**: Crank up `noise_amplitude` to 1×10⁻⁵, run several times → all trajectories converge to the same state
2. **Find the safety margin**: Click "Compute Bifurcation Diagram" → measure ΔT distance to the nearest tipping diamond
3. **Slowly warm the pole**: Inch T₂ upward while running → watch q_sv drop in the AMOC bar chart

## Preset 2: Weakened AMOC

**Backend name**: `weakened_amoc`
**Description**: Reduced thermal forcing — circulation noticeably weaker.

### Parameters

| Parameter | Value |
|-----------|-------|
| T₁ | 25 °C |
| **T₂** | **12 °C** |
| **ΔT** | **13 °C** |
| S₁ᵉᵍ | 36 psu |
| S₂ᵉᵍ | 34 psu |
| S₁ᵢⁿⁱᵗ | 36 psu |
| S₂ᵢⁿⁱᵗ | 34 psu |

### What to Observe

**Time series**:
- Slower equilibration (the "critical slowing down" signal)
- Final q_sv noticeably reduced from Normal AMOC
- Salinity gradient shrinks compared to Normal

**Phase portrait**:
- Trajectory takes longer to settle; the attractor is closer to the unstable middle branch

**Two-box diagram**:
- Thinner arrow

**AMOC comparison**:
- Bar is well below the RAPID observed mean

**Bifurcation diagram**:
- Orange X marker is at ΔT = 13 °C, much closer to the collapse diamond
- May be inside or near the bistable window depending on chosen parameters

### Physical Interpretation

A **near-future projection** of the AMOC under polar amplification. Polar warming has shrunk ΔT, weakening the thermally-driven flow.

### Experiments to Try

1. **Find the tipping point**: Inch T₂ up further (12 → 14 → 16) and watch q crash
2. **Stochastic tipping**: Set noise_amplitude to 3×10⁻⁵ and run 5–10 times — some runs will collapse, some won't
3. **Compute bifurcation**: See exactly where ΔT_crit sits

## Preset 3: Collapsed AMOC

**Backend name**: `collapsed_amoc`
**Description**: Near-collapsed state. ΔT is very small and the polar box starts fresher (lower salinity gradient).

### Parameters

| Parameter | Value |
|-----------|-------|
| T₁ | 25 °C |
| **T₂** | **20 °C** |
| **ΔT** | **5 °C** |
| S₁ᵉᵍ | 36 psu |
| S₂ᵉᵍ | 34 psu |
| S₁ᵢⁿⁱᵗ | 36 psu |
| **S₂ᵢⁿⁱᵗ** | **36.5 psu** (saltier than equator → reversed gradient) |

### What to Observe

**Time series**:
- q_sv ends up near zero or slightly negative (haline-driven reversal)
- System never recovers a strong AMOC

**Phase portrait**:
- Trajectory pulled toward the lower (collapsed/haline) branch

**Two-box diagram**:
- Arrow is small or reversed (q ≤ 0)

**AMOC comparison**:
- Bar is near zero; dramatically below the observed value

**Bifurcation diagram**:
- Orange X is at ΔT = 5 °C, well to the left of the collapse diamond — only the lower branch exists

### Physical Interpretation

A **collapsed AMOC state**, qualitatively similar to:
- Inferred glacial / Heinrich-event circulation
- Worst-case future scenario after a tipping event

The thermal drive is so weak (ΔT = 5 °C) that the haline mode dominates and the upper branch doesn't exist as a stable attractor.

### Experiments to Try

1. **Try to recover**: Drag T₂ back down to 5 °C → run → does it recover, or is it stuck? (Initial conditions matter near tipping.)
2. **Cold start**: Set S₂ᵢⁿⁱᵗ back to 34 and re-run → see whether different initial conditions give different fates
3. **Compare with Normal**: Load Normal AMOC, compute bifurcation, then load Collapsed → visualize they sit on different branches

## Preset 4: Stommel 1961 Original

**Backend name**: `stommel_original`
**Description**: Stommel's 1961 parameter values with moderate thermal forcing (ΔT = 17.5 °C).

### Parameters

| Parameter | Value |
|-----------|-------|
| T₁ | 25 °C |
| **T₂** | **7.5 °C** |
| **ΔT** | **17.5 °C** |
| S₁ᵉᵍ | 36 psu |
| S₂ᵉᵍ | 34 psu |
| S₁ᵢⁿⁱᵗ | 36 psu |
| S₂ᵢⁿⁱᵗ | 34 psu |

### What to Observe

**Time series**:
- Intermediate equilibration time
- Final q_sv between Normal and Weakened — moderate but stable

**Bifurcation diagram**:
- Orange X is at ΔT = 17.5 °C, often **inside the bistable region** depending on parameters
- The classic regime in which Stommel originally demonstrated bistability and hysteresis

### Physical Interpretation

The setup Stommel used in his 1961 *Tellus* paper to demonstrate that two stable circulation regimes can coexist. Use this preset to reproduce the famous S-curve.

### Experiments to Try

1. **Reproduce Stommel's S-curve**: Click Compute Bifurcation Diagram → see the classic fold
2. **Demonstrate hysteresis**:
   - Start here → run → note upper branch
   - Drag T₂ up to 20 °C → run → collapse to lower branch
   - Drag T₂ back to 7.5 °C → run → still on the lower branch (hysteresis!)
   - Drag T₂ even lower (e.g. 0 °C) → run → AMOC suddenly recovers
3. **Read the original**: See [Stommel 1961](/theory/stommel-1961) and [Bifurcations & Hysteresis](/theory/bifurcations-hysteresis)

## Comparison: Presets at a Glance

| Aspect | Normal | Weakened | Collapsed | Stommel 1961 |
|--------|--------|----------|-----------|--------------|
| **T₂ (°C)** | 5 | 12 | 20 | 7.5 |
| **ΔT (°C)** | 20 | 13 | 5 | 17.5 |
| **S₂ᵢⁿⁱᵗ (psu)** | 34 | 34 | 36.5 | 34 |
| **Final q_sv** | ~15 Sv | weakened | ~0 Sv | intermediate |
| **Regime** | Strong, stable | Marginal | Collapsed (haline) | Bistable, classic |
| **In bifurcation diagram** | Right (monostable strong) | Near collapse | Left (monostable weak) | Inside bistable window |
| **Resembles** | Present AMOC | Near-future risk | Glacial / worst-case | Stommel's textbook setup |

## Using Presets Effectively

### Quick Exploration

1. Click a preset name → all sliders snap to that preset's values
2. Click **Run Simulation**
3. Observe the 5 visualization panels populate
4. Compare to other presets

### Systematic Study

1. Load **Normal AMOC** → run → note equilibrium q_sv
2. Load **Weakened** → run → observe gradual weakening
3. Load **Collapsed** → run → see near-zero circulation
4. Try to recover from collapsed: lower T₂ slider manually and re-run

### Bifurcation Analysis

1. Load **Stommel 1961**
2. Click **Compute Bifurcation Diagram**
3. Observe the S-curve with hysteresis window shaded
4. Load **Normal AMOC** and re-compute → see how the curve looks at different starting ΔT

## Can I Modify a Preset?

Yes! Load a preset, then:
- Adjust any sliders
- Click **Run Simulation** again — it uses your modified parameters
- To revert to the preset's original values, click its button again
- To restore the **factory defaults** (the StommelModel dataclass defaults), click the red **Reset** button

## Loading Presets via the API

The presets endpoint returns a JSON object wrapping a `presets` list. Each preset has `name`, `description`, and `params`:

```bash
curl -s http://localhost:8000/api/presets | jq '.presets[].name'
# "normal_amoc"
# "weakened_amoc"
# "collapsed_amoc"
# "stommel_original"
```

See [GET /api/presets](/api-reference/presets) for the full schema.
