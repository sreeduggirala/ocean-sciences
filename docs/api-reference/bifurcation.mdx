# POST /bifurcation

Compute a **bifurcation diagram** by sweeping freshwater forcing (F) in both directions (forward and backward) and tracking equilibrium circulation strength.

This is the endpoint used by the "Compute Bifurcation" button in the UI.

---

## Endpoint

```
POST /bifurcation
```

---

## Request

### Headers

```
Content-Type: application/json
```

### Body (JSON)

```json
{
  "T_e": 25,
  "T_p": 5,
  "S_e0": 36,
  "S_p0": 34,
  "alpha": 1.5e-4,
  "beta": 8e-4,
  "k": 1.5e-6,
  "F_min": 0,
  "F_max": 5e-4,
  "n_points": 200
}
```

### Parameter Descriptions

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| T_e | float | 25 | Equatorial temperature (°C) |
| T_p | float | 5 | Polar temperature (°C) |
| S_e0 | float | 36 | Initial equatorial salinity (psu) |
| S_p0 | float | 34 | Initial polar salinity (psu) |
| alpha | float | 1.5e-4 | Thermal expansion (1/°C) |
| beta | float | 8e-4 | Haline contraction (1/psu) |
| k | float | 1.5e-6 | Circulation coefficient (1/s) |
| **F_min** | float | 0 | Minimum freshwater flux (psu/s) |
| **F_max** | float | 5e-4 | Maximum freshwater flux (psu/s) |
| **n_points** | int | 200 | Number of F values to sample |

### Algorithm

1. **Forward sweep** (F increasing):
   - Sample F from F_min to F_max in n_points equally-spaced steps
   - For each F, run simulation with t_max=100 years
   - Track equilibrium circulation q_sv_final
   - Returns: `q_forward` array

2. **Backward sweep** (F decreasing):
   - Sample F from F_max to F_min in n_points equally-spaced steps
   - For each F, run simulation with t_max=100 years
   - Returns: `q_backward` array

3. **Bifurcation point detection**:
   - Identify sudden jumps in q between adjacent F values (magnitude > 2×10⁻⁷ 1/s)
   - Mark these as tipping points

---

## Response

### Success (200 OK)

```json
{
  "parameters": {
    "T_e": 25,
    "T_p": 5,
    "S_e0": 36,
    "S_p0": 34,
    "alpha": 1.5e-4,
    "beta": 8e-4,
    "k": 1.5e-6,
    "F_min": 0,
    "F_max": 5e-4,
    "n_points": 200
  },
  "F_values": [0, 2.5e-5, 5e-5, ..., 5e-4],
  "q_forward": [2.5e-9, 2.4e-9, 2.3e-9, ..., -1e-10],
  "q_backward": [2.5e-9, 2.4e-9, 2.3e-9, ..., -5e-11],
  "q_sv_forward": [17.5, 17.3, 17.1, ..., -0.7],
  "q_sv_backward": [17.5, 17.3, 17.1, ..., -0.35],
  "tipping_points": {
    "F_c1": 2.0e-4,
    "F_c2": 1.0e-4,
    "q_at_F_c1": 2.1e-9,
    "q_at_F_c2": 2.0e-9
  },
  "computation_time_sec": 25.4
}
```

### Field Descriptions

| Field | Type | Description |
|-------|------|-------------|
| parameters | object | Echo of input parameters |
| F_values | array | F points sampled (length = n_points) |
| q_forward | array | Equilibrium q (1/s) for forward sweep |
| q_backward | array | Equilibrium q (1/s) for backward sweep |
| q_sv_forward | array | Same as q_forward but in Sverdrups |
| q_sv_backward | array | Same as q_backward but in Sverdrups |
| tipping_points | object | Detected bifurcation points |
| tipping_points.F_c1 | float | Forward tipping point (collapse threshold) |
| tipping_points.F_c2 | float | Backward tipping point (recovery threshold) |
| tipping_points.q_at_F_c1 | float | q value at collapse point (1/s) |
| tipping_points.q_at_F_c2 | float | q value at recovery point (1/s) |
| computation_time_sec | float | Total wall-clock time |

### Interpretation

- **F_c1**: Freshwater flux where upper branch ends; collapse occurs beyond this
- **F_c2**: Freshwater flux where lower branch ends; recovery occurs beyond this
- **Hysteresis width**: F_c1 − F_c2 (F range where two states coexist)
- **Forward/backward**: Show both branches; difference reveals hysteresis

## Examples

### Minimal Request (Default Parameters)

```bash
curl -X POST http://localhost:8000/bifurcation \
  -H "Content-Type: application/json" \
  -d '{}'
```

Uses default parameters and 200 F points.

### Custom F Range and Resolution

```bash
curl -X POST http://localhost:8000/bifurcation \
  -H "Content-Type: application/json" \
  -d '{
    "F_min": 0.5e-4,
    "F_max": 3e-4,
    "n_points": 100
  }'
```

Focuses on the bistable region with coarser resolution (faster).

### Python Client Example

```python
import requests
import matplotlib.pyplot as plt
import numpy as np

# Request bifurcation diagram
response = requests.post('http://localhost:8000/bifurcation', json={
    "F_min": 0,
    "F_max": 5e-4,
    "n_points": 200
})

data = response.json()

# Extract results
F = np.array(data['F_values'])
q_fwd = np.array(data['q_sv_forward'])
q_bwd = np.array(data['q_sv_backward'])
F_c1 = data['tipping_points']['F_c1']
F_c2 = data['tipping_points']['F_c2']

# Plot S-curve
fig, ax = plt.subplots(figsize=(10, 6))

ax.plot(F*1e4, q_fwd, 'b-', linewidth=2, label='Forward (increasing F)')
ax.plot(F*1e4, q_bwd, 'o-', color='orange', linewidth=2, label='Backward (decreasing F)')

# Mark bifurcation points
ax.plot(F_c1*1e4, data['tipping_points']['q_at_F_c1']*7e15/1e6, 'r*',
        markersize=15, label=f'F_c1 = {F_c1:.1e}')
ax.plot(F_c2*1e4, data['tipping_points']['q_at_F_c2']*7e15/1e6, 'rs',
        markersize=10, label=f'F_c2 = {F_c2:.1e}')

# Shade bistable region
ax.axvspan(F_c2*1e4, F_c1*1e4, alpha=0.2, color='yellow', label='Bistable region')

ax.set_xlabel('Freshwater Flux (×10⁻⁴ psu/s)')
ax.set_ylabel('Circulation (Sv)')
ax.set_title('Bifurcation Diagram (S-Curve)')
ax.legend()
ax.grid(True, alpha=0.3)

plt.tight_layout()
plt.savefig('bifurcation.png', dpi=150)
plt.show()

print(f"Hysteresis width: {(F_c1 - F_c2)*1e4:.2f} × 10⁻⁴ psu/s")
```

## Common Use Cases

### 1. Analyze Sensitivity to Temperature

```python
# Baseline
resp1 = requests.post('http://localhost:8000/bifurcation',
                      json={"T_p": 5})
data1 = resp1.json()
F_c1_baseline = data1['tipping_points']['F_c1']

# Warmer pole (polar amplification)
resp2 = requests.post('http://localhost:8000/bifurcation',
                      json={"T_p": 8})
data2 = resp2.json()
F_c1_warm = data2['tipping_points']['F_c1']

print(f"ΔT=20°C: F_c1 = {F_c1_baseline:.1e}")
print(f"ΔT=17°C: F_c1 = {F_c1_warm:.1e}")
print(f"Effect: Warming pole lowers critical F by {(F_c1_baseline-F_c1_warm)*1e4:.2f}×10⁻⁴")
```

### 2. Compare Parameter Sets

```python
scenarios = {
    "Current": {"alpha": 1.5e-4, "beta": 8e-4},
    "High α": {"alpha": 2.5e-4, "beta": 8e-4},
    "High β": {"alpha": 1.5e-4, "beta": 1.2e-3},
}

results = {}
for name, params in scenarios.items():
    resp = requests.post('http://localhost:8000/bifurcation', json=params)
    data = resp.json()
    F_c1 = data['tipping_points']['F_c1']
    hysteresis = F_c1 - data['tipping_points']['F_c2']
    results[name] = {"F_c1": F_c1, "hysteresis": hysteresis}

for name, vals in results.items():
    print(f"{name}: F_c1={vals['F_c1']:.1e}, width={vals['hysteresis']:.1e}")
```

### 3. Fine-Tune Around Tipping Point

```python
# Coarse sweep to find region
resp = requests.post('http://localhost:8000/bifurcation',
                     json={"n_points": 50})
data = resp.json()
F_c1_approx = data['tipping_points']['F_c1']

# Fine sweep around tipping point
resp_fine = requests.post('http://localhost:8000/bifurcation',
                          json={
                              "F_min": F_c1_approx - 1e-4,
                              "F_max": F_c1_approx + 1e-4,
                              "n_points": 200
                          })
data_fine = resp_fine.json()
F_c1_precise = data_fine['tipping_points']['F_c1']

print(f"Refined F_c1: {F_c1_precise:.2e} psu/s")
```

## Performance Characteristics

| n_points | # Simulations | Time (typical) | Use Case |
|----------|---------------|----------------|----------|
| 50 | 100 | ~5 sec | Quick exploration |
| 100 | 200 | ~10 sec | Standard |
| 200 | 400 | ~20–30 sec | Publication quality |
| 500 | 1000 | ~60+ sec | Very fine detail |

Computation time scales roughly linearly with n_points. Each simulation takes ~50–100 ms.

---

## Output Units

**Important**: The q values in the response are in **1/s** (dimensionless), not Sverdrups.

To convert:
```
q_sv = q × (box_volume) / 10^6
     = q × (7×10^15 m³) / 10^6
     ≈ q × 7×10^9
```

The response includes both `q_forward/q_backward` (raw) and `q_sv_forward/q_sv_backward` (converted).

---

## Bifurcation Detection Algorithm

Tipping points are detected by:

1. Computing the magnitude of jumps between adjacent F values
2. If |q(F_i+1) − q(F_i)| > **2×10⁻⁷** (1/s), flagging as a potential bifurcation
3. Refining to the exact F value by bisection search

**Note**: This is a heuristic algorithm. Very close bifurcations or smooth transitions near criticality may not be precisely detected. For publication-quality bifurcation diagrams, increasing n_points improves precision.

## Error Handling

### Invalid Parameters

```bash
curl -X POST http://localhost:8000/bifurcation \
  -H "Content-Type: application/json" \
  -d '{"T_e": 10, "T_p": 20}'
```

Response (422):
```json
{"detail": [{"type": "value_error", "msg": "T_e must be greater than T_p"}]}
```

### F_min >= F_max

```bash
curl -X POST http://localhost:8000/bifurcation \
  -H "Content-Type: application/json" \
  -d '{"F_min": 3e-4, "F_max": 1e-4}'
```

Response (typically 200, but bifurcation diagram may be degenerate):
```json
{
  "parameters": {...},
  "F_values": [3e-4, 2.5e-4, ..., 1e-4],
  "q_forward": [...],
  "q_backward": [...],
  "tipping_points": null
}
```
