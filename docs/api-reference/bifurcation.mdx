# POST /api/bifurcation

Compute a **bifurcation diagram** by sweeping the polar temperature `T_2` in both directions (forward and backward) and recording the steady-state circulation strength at each value.

This is the endpoint invoked by the **Compute Bifurcation Diagram** button in the UI.

> The bifurcation parameter in the original Stommel (1961) relaxation model is the temperature difference **ΔT = T₁ − T₂**. The endpoint sweeps `T_2` while holding `T_1` fixed, which is equivalent to sweeping ΔT.

---

## Endpoint

```
POST /api/bifurcation
```

---

## Request

### Headers

```
Content-Type: application/json
```

### Body (JSON)

All fields are optional.

```json
{
  "T_1": 25.0,
  "T_2": 5.0,
  "S_1_eq": 36.0,
  "S_2_eq": 34.0,
  "alpha": 2.0e-4,
  "beta": 1.0e-3,
  "k": 3.0e-9,
  "lam": 3.0e-11,
  "T_2_min": 0.0,
  "T_2_max": 20.0,
  "n_points": 200
}
```

### Field Reference

| Field | Type | Default | Units | Range | Description |
|-------|------|---------|-------|-------|-------------|
| `T_1` | float | 25.0 | °C | 15 – 35 | Box 1 temperature (held fixed during sweep) |
| `T_2` | float | 5.0 | °C | -2 – 25 | Initial T₂ (only used to seed the model) |
| `S_1_eq` | float | 36.0 | psu | 30 – 40 | Box 1 relaxation target |
| `S_2_eq` | float | 34.0 | psu | 28 – 40 | Box 2 relaxation target |
| `alpha` | float | 2.0e-4 | 1/°C | 0.5e-4 – 3e-4 | Thermal expansion |
| `beta` | float | 1.0e-3 | 1/psu | 2e-4 – 2e-3 | Haline contraction |
| `k` | float | 3.0e-9 | 1/s | 1e-10 – 1e-6 | Circulation coefficient |
| `lam` | float | 3.0e-11 | 1/s | 1e-12 – 1e-9 | Salinity relaxation rate |
| **`T_2_min`** | float | 0.0 | °C | -5 – 25 | Minimum T₂ in the sweep |
| **`T_2_max`** | float | 20.0 | °C | 0 – 30 | Maximum T₂ in the sweep |
| **`n_points`** | int | 200 | – | 10 – 500 | Number of T₂ samples (used for *each* of the forward and backward sweeps) |

### Validation Rules

- `T_2_min < T_2_max` enforced
- `T_1 > T_2_min` enforced
- All `ge`/`le` bounds enforced

### Algorithm

1. Build `n_points` values of `T_2` linearly spaced in `[T_2_min, T_2_max]`.
2. **Forward sweep** (T₂ ↑, ΔT ↓):
   - Start state: `[S_1_eq, S_2_eq]` (relaxation equilibrium, near upper branch)
   - For each T₂ in increasing order: build a fresh `StommelModel`, integrate for 100 years, record the final `q`, then carry the final state into the next iteration as the new initial condition
3. **Backward sweep** (T₂ ↓, ΔT ↑):
   - Start state: `[S_1_eq, 38.0]` (very fresh polar box → forces lower / haline branch)
   - For each T₂ in decreasing order: same procedure, results stored at the corresponding original index
4. **Tipping-point detection**: scan adjacent `q` values; flag any jump with `|Δq| > 2 × 10⁻⁷ 1/s` as a tipping point.

This continuation strategy is what reveals hysteresis: forward sweep traces the upper branch until it dies, backward sweep traces the lower branch.

---

## Response

### Success (200 OK)

```json
{
  "T_2_values": [0.0, 0.10, 0.20, ..., 20.0],
  "delta_T_values": [25.0, 24.90, 24.80, ..., 5.0],
  "q_forward": [3.5e-9, 3.48e-9, ..., 0.1e-9],
  "q_backward": [3.5e-9, 3.48e-9, ..., 0.05e-9],
  "tipping_points": [
    {"T_2": 14.5, "delta_T": 10.5, "q": 0.2e-9, "direction": "forward"},
    {"T_2": 3.0,  "delta_T": 22.0, "q": 3.0e-9, "direction": "backward"}
  ]
}
```

### Field Reference

| Field | Type | Description |
|-------|------|-------------|
| `T_2_values` | array of float | The `n_points` T₂ values swept (°C, increasing order) |
| `delta_T_values` | array of float | `T_1 − T_2_values` (°C, decreasing order) |
| `q_forward` | array of float | Steady-state circulation `q` (1/s) from the forward (T₂ ↑) sweep at each `T_2` |
| `q_backward` | array of float | Steady-state circulation `q` (1/s) from the backward (T₂ ↓) sweep at each `T_2` |
| `tipping_points` | array of object | Detected jumps. Each item has `T_2`, `delta_T`, `q`, and `direction` ∈ `"forward"` / `"backward"` |

> The response contains `q` in **1/s only**. To convert to Sverdrups multiply by `V_BOX / 1e6 = 7×10⁹`. There are no `q_sv_forward` / `q_sv_backward` fields, no `F_values`, no `F_c1` / `F_c2`, and no `computation_time_sec` — the older API docs were wrong about all of these.

## Examples

### Minimal request (defaults)

```bash
curl -X POST http://localhost:8000/api/bifurcation \
  -H "Content-Type: application/json" \
  -d '{}'
```

### Custom sweep range and resolution

```bash
curl -X POST http://localhost:8000/api/bifurcation \
  -H "Content-Type: application/json" \
  -d '{
    "T_2_min": 5,
    "T_2_max": 18,
    "n_points": 100
  }'
```

### Python — plot the S-curve

```python
import requests
import numpy as np
import matplotlib.pyplot as plt

V_BOX = 7e15  # m^3, Sv conversion
SV = V_BOX / 1e6

r = requests.post('http://localhost:8000/api/bifurcation', json={
    "T_2_min": 0,
    "T_2_max": 20,
    "n_points": 200,
})
data = r.json()

dT = np.array(data['delta_T_values'])
q_fwd = np.array(data['q_forward']) * SV
q_bwd = np.array(data['q_backward']) * SV

fig, ax = plt.subplots(figsize=(10, 6))
ax.plot(dT, q_fwd, 'b-', linewidth=2, label='Forward sweep (T₂↑, ΔT↓)')
ax.plot(dT, q_bwd, 'o-', color='orange', linewidth=2, label='Backward sweep (T₂↓, ΔT↑)')

for tp in data['tipping_points']:
    ax.plot(tp['delta_T'], tp['q'] * SV, 'r*', markersize=15)

ax.set_xlabel('ΔT = T₁ − T₂ (°C)')
ax.set_ylabel('Circulation q (Sv)')
ax.set_title('Stommel S-Curve (relaxation model)')
ax.invert_xaxis()  # Forward sweep direction
ax.legend(); ax.grid(alpha=0.3)
plt.tight_layout()
plt.savefig('bifurcation.png', dpi=150)
plt.show()
```

### JavaScript / fetch

```javascript
const r = await fetch('http://localhost:8000/api/bifurcation', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    T_2_min: 0, T_2_max: 20, n_points: 200,
  })
});
const data = await r.json();

const SV = 7e9;
const dT = data.delta_T_values;
const q_fwd_sv = data.q_forward.map(q => q * SV);
const q_bwd_sv = data.q_backward.map(q => q * SV);

console.log(`Tipping points: ${data.tipping_points.length}`);
data.tipping_points.forEach(tp => {
  console.log(`  ${tp.direction} jump at ΔT=${tp.delta_T.toFixed(1)}°C, q=${(tp.q*SV).toFixed(2)} Sv`);
});
```

## Common Use Cases

### 1. Sensitivity to thermal expansion

```python
results = {}
for alpha in [1.0e-4, 2.0e-4, 3.0e-4]:
    r = requests.post('http://localhost:8000/api/bifurcation',
                      json={"alpha": alpha})
    data = r.json()
    fwd_tips = [tp for tp in data['tipping_points'] if tp['direction']=='forward']
    if fwd_tips:
        results[alpha] = fwd_tips[0]['delta_T']

for alpha, dT_crit in results.items():
    print(f"α={alpha:.1e} → ΔT_crit ≈ {dT_crit:.2f}°C")
```

### 2. Compare presets

```python
# Fetch presets, then compute bifurcation for each
presets = requests.get('http://localhost:8000/api/presets').json()['presets']

for p in presets:
    bif_params = {
        k: v for k, v in p['params'].items()
        if k in {'T_1', 'S_1_eq', 'S_2_eq', 'alpha', 'beta', 'k', 'lam'}
    }
    r = requests.post('http://localhost:8000/api/bifurcation', json=bif_params)
    n_tips = len(r.json()['tipping_points'])
    print(f"{p['name']:>20}: {n_tips} tipping points detected")
```

### 3. Refine around the collapse threshold

```python
# Coarse scan
r = requests.post('http://localhost:8000/api/bifurcation',
                  json={"n_points": 50})
data = r.json()
fwd_tips = [tp for tp in data['tipping_points'] if tp['direction']=='forward']
if fwd_tips:
    T2_approx = fwd_tips[0]['T_2']
    # Fine scan
    r2 = requests.post('http://localhost:8000/api/bifurcation', json={
        "T_2_min": T2_approx - 1.0,
        "T_2_max": T2_approx + 1.0,
        "n_points": 200,
    })
    print("Refined collapse at T_2 =", r2.json()['tipping_points'][0]['T_2'])
```

## Performance

| `n_points` | # mini-simulations | Wall time (typical) |
|-----------:|-------------------:|--------------------:|
| 50 | 100 | ~3 s |
| 100 | 200 | ~6 s |
| 200 (default) | 400 | ~12 s |
| 500 | 1000 | ~30+ s |

Each mini-simulation uses `t_max = 100` years (hard-coded in `solver.compute_bifurcation`), which is enough to reach steady state at any single ΔT.

The computation runs in a `ThreadPoolExecutor` so it doesn't block the FastAPI event loop.

---

## Tipping-Point Detection

The threshold `2 × 10⁻⁷ 1/s` (≈ 1.4 Sv) is heuristic. Effects:

- True saddle-node bifurcations create gigantic jumps and are reliably caught
- Smooth transitions or very slow drifts are not flagged
- Increase `n_points` to bracket the jump more precisely

If you need exact threshold values, post-process the `q_forward` / `q_backward` arrays yourself (e.g. find the largest `|np.diff(q)|` index).
