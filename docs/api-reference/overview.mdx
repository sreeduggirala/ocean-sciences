# API Reference Overview

## Base URL

When running the backend locally with `uvicorn api:app --reload`, all API endpoints are mounted under the **`/api`** prefix:

```
http://localhost:8000/api
```

When running through the Vite dev server (`npm run dev` on port 5173), Vite proxies `/api/*` to the backend, so the frontend uses paths like `/api/simulate` directly.

In production deployments where the backend serves the built frontend on the same origin, the same `/api/*` paths apply.

---

## Architecture Overview

The backend is built with **FastAPI** and exposes:

- **RESTful endpoints** for simulations and bifurcation analysis
- **Pydantic v2 validation** on every request body
- **async/await** non-blocking I/O
- **`ThreadPoolExecutor(max_workers=2)`** so long-running simulations don't block the event loop

The simulator is implemented in `backend/`:
- `model.py` — the `StommelModel` dataclass with `q()` and `rhs_years()` for ODEs
- `solver.py` — `run_simulation()` (RK45 deterministic + Euler-Maruyama stochastic) and `compute_bifurcation()` (forward + backward T₂ sweep)
- `schemas.py` — Pydantic request/response models
- `api.py` — FastAPI app

---

## Endpoints Summary

| Endpoint | Method | Purpose |
|----------|--------|---------|
| `/api/health` | GET | Health check |
| `/api/simulate` | POST | Run a single simulation |
| `/api/bifurcation` | POST | Compute the bifurcation S-curve (sweeps T₂) |
| `/api/presets` | GET | List the four named preset configurations |

---

## General Principles

### Request Format

POST endpoints accept JSON bodies:

```bash
curl -X POST http://localhost:8000/api/simulate \
  -H "Content-Type: application/json" \
  -d '{"T_1": 25, "T_2": 5}'
```

GET endpoints take no body and no query parameters in this version.

### Response Format

All responses are JSON. Schemas are defined in `backend/schemas.py`. See the individual endpoint pages for full field listings.

### HTTP Status Codes

| Code | Meaning | Example |
|------|---------|---------|
| 200 | Success | Simulation completed |
| 422 | Validation Error | T₁ ≤ T₂, or parameter out of range |
| 500 | Server Error | Unhandled exception (rare) |

---

## Authentication & CORS

### No Authentication

All endpoints are open. If you deploy publicly, add an auth layer in front.

### CORS

CORS is configured in `api.py` from the `CORS_ALLOW_ORIGINS` env var, defaulting to:
- `http://localhost:5173` (Vite dev server)
- `http://localhost:3000` (alternative)

Override with `CORS_ALLOW_ORIGINS="https://your-domain.example,..."` when deploying.

## Common Parameters

These are the parameters that appear in both `/api/simulate` and `/api/bifurcation` requests. **All have defaults**, so you can send `{}` and get a working simulation.

| Parameter | Type | Default | Units | Range | Description |
|-----------|------|---------|-------|-------|-------------|
| `T_1` | float | 25.0 | °C | 15 – 35 | Box 1 (equatorial) temperature |
| `T_2` | float | 5.0 | °C | -2 – 25 | Box 2 (polar) temperature |
| `S_1_eq` | float | 36.0 | psu | 30 – 40 | Box 1 equilibrium (relaxation target) salinity |
| `S_2_eq` | float | 34.0 | psu | 28 – 40 | Box 2 equilibrium salinity |
| `alpha` | float | 2.0e-4 | 1/°C | 0.5e-4 – 3e-4 | Thermal expansion (Stommel original) |
| `beta` | float | 1.0e-3 | 1/psu | 2e-4 – 2e-3 | Haline contraction (Stommel original) |
| `k` | float | 3.0e-9 | 1/s | 1e-10 – 1e-6 | Circulation coefficient (Stommel original) |
| `lam` | float | 3.0e-11 | 1/s | 1e-12 – 1e-9 | Salinity relaxation rate (~1000-yr timescale) |

`/api/simulate` additionally takes:

| Parameter | Type | Default | Units | Range |
|-----------|------|---------|-------|-------|
| `S_1_init` | float | 36.0 | psu | 30 – 40 |
| `S_2_init` | float | 34.0 | psu | 28 – 40 |
| `t_max` | float | 3000 | years | 100 – 10000 |
| `dt` | float | 1.0 | years | 0.1 – 100 |
| `noise_amplitude` | float | 0.0 | psu/s | 0 – 5e-5 |

`/api/bifurcation` additionally takes:

| Parameter | Type | Default | Units | Range |
|-----------|------|---------|-------|-------|
| `T_2_min` | float | 0.0 | °C | -5 – 25 |
| `T_2_max` | float | 20.0 | °C | 0 – 30 |
| `n_points` | int | 200 | – | 10 – 500 |

### Validation Rules

- All `ge`/`le` bounds are enforced by Pydantic
- `T_1 > T_2` is enforced (custom validator)
- For bifurcation requests: `T_2_min < T_2_max` and `T_1 > T_2_min`

> **Note:** This is the **original Stommel (1961) relaxation model** — there is no freshwater flux `F` parameter. The bifurcation parameter is the temperature difference ΔT = T₁ − T₂, which the bifurcation endpoint sweeps by varying T₂.

## Error Handling

### Validation Error (422)

```bash
curl -X POST http://localhost:8000/api/simulate \
  -H "Content-Type: application/json" \
  -d '{"T_1": 5, "T_2": 25}'
```

```json
{
  "detail": [
    {
      "type": "value_error",
      "loc": ["body"],
      "msg": "Value error, T_1 must be greater than T_2"
    }
  ]
}
```

### Out-of-bounds parameter

```bash
curl -X POST http://localhost:8000/api/simulate \
  -H "Content-Type: application/json" \
  -d '{"alpha": 1e-2}'
```

```json
{
  "detail": [{
    "type": "less_than_equal",
    "loc": ["body", "alpha"],
    "msg": "Input should be less than or equal to 0.0003"
  }]
}
```

### Server Error (500)

Rare; check the backend terminal for the traceback.

## Performance & Timeouts

### Computation Time (typical)

| Scenario | Wall time |
|----------|-----------|
| Single deterministic simulation (`t_max=3000`) | ~0.1–1 s |
| Single stochastic simulation (`t_max=3000`, dt=1) | ~0.5–2 s |
| Bifurcation diagram (`n_points=200`, two sweeps × 100-yr each) | ~10–30 s |

`ThreadPoolExecutor(max_workers=2)` queues concurrent requests.

There is no application-level HTTP timeout; uvicorn's default applies.

### Memory

- Single simulation response: ~50–500 KB JSON depending on `t_max/dt`
- Bifurcation response: ~5 KB JSON (only steady-state q values, not full trajectories)

## Example: Complete Request-Response Cycle

**1. Check server health**
```bash
curl http://localhost:8000/api/health
# {"status": "ok"}
```

**2. Run simulation with defaults**
```bash
curl -X POST http://localhost:8000/api/simulate \
  -H "Content-Type: application/json" \
  -d '{}'
```

Response (truncated):
```json
{
  "time": [0, 1, 2, ..., 3000],
  "S_1": [36.0, ...],
  "S_2": [34.0, ...],
  "q": [3.0e-9, ...],
  "q_sv": [21.0, ...],
  "steady_state_reached": true,
  "final_state": [36.18, 33.82],
  "metadata": {"t_max_years": 3000, "noise_amplitude": 0.0}
}
```

**3. Read final AMOC strength in Python**
```python
import requests
r = requests.post('http://localhost:8000/api/simulate', json={})
data = r.json()
print(f"Final AMOC: {data['q_sv'][-1]:.1f} Sv")
print(f"Steady state: {data['steady_state_reached']}")
```

## Debugging

### Enable verbose backend logging

```bash
uv run uvicorn api:app --reload --log-level debug
```

### Use a REST client

Postman, Insomnia, Bruno, or VS Code's "Thunder Client" / REST Client extensions all work.

### Inspect from the browser

Open the simulator at http://localhost:5173, open DevTools → Network tab, watch the `/api/*` calls go by, and click any of them to see the exact request/response payloads.

---

## Production Considerations

If you deploy publicly:

1. **Authentication** (API keys / OAuth)
2. **Rate limiting** (per-IP)
3. **HTTPS** (TLS termination at a reverse proxy)
4. **Tighten CORS** via `CORS_ALLOW_ORIGINS`
5. **Logging & monitoring** (request logs, error rates)
6. **Caching** for `/api/bifurcation` since it is deterministic and slow
