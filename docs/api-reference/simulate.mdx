# POST /api/simulate

Integrate the Stommel two-box thermohaline circulation model forward in time and return the full trajectory.

---

## Endpoint

```
POST /api/simulate
```

---

## Request

### Headers

```
Content-Type: application/json
```

### Body (JSON)

All fields are **optional** — defaults match Stommel's 1961 original parameters. Sending `{}` runs the canonical baseline simulation.

```json
{
  "T_1": 25.0,
  "T_2": 5.0,
  "S_1_eq": 36.0,
  "S_2_eq": 34.0,
  "S_1_init": 36.0,
  "S_2_init": 34.0,
  "alpha": 2.0e-4,
  "beta": 1.0e-3,
  "k": 3.0e-9,
  "lam": 3.0e-11,
  "t_max": 3000,
  "dt": 1.0,
  "noise_amplitude": 0.0
}
```

### Field Reference

| Field | Type | Default | Units | Range | Description |
|-------|------|---------|-------|-------|-------------|
| `T_1` | float | 25.0 | °C | 15 – 35 | Box 1 (equatorial) temperature |
| `T_2` | float | 5.0 | °C | -2 – 25 | Box 2 (polar) temperature |
| `S_1_eq` | float | 36.0 | psu | 30 – 40 | Box 1 relaxation-target salinity |
| `S_2_eq` | float | 34.0 | psu | 28 – 40 | Box 2 relaxation-target salinity |
| `S_1_init` | float | 36.0 | psu | 30 – 40 | Box 1 initial salinity (state at t = 0) |
| `S_2_init` | float | 34.0 | psu | 28 – 40 | Box 2 initial salinity (state at t = 0) |
| `alpha` | float | 2.0e-4 | 1/°C | 0.5e-4 – 3e-4 | Thermal expansion |
| `beta` | float | 1.0e-3 | 1/psu | 2e-4 – 2e-3 | Haline contraction |
| `k` | float | 3.0e-9 | 1/s | 1e-10 – 1e-6 | Circulation coefficient |
| `lam` | float | 3.0e-11 | 1/s | 1e-12 – 1e-9 | Salinity relaxation rate (~1000-yr timescale) |
| `t_max` | float | 3000 | years | 100 – 10000 | Total integration time |
| `dt` | float | 1.0 | years | 0.1 – 100 | Output sampling interval |
| `noise_amplitude` | float | 0.0 | psu/s | 0 – 5e-5 | Gaussian noise std (0 = deterministic) |

### Validation Rules

- All `ge`/`le` bounds enforced
- `T_1 > T_2` enforced (custom validator)
- Otherwise no constraints — total salt is **not** conserved (relaxation model)

## Response

### Success (200 OK)

```json
{
  "time": [0, 1, 2, 3, ...],
  "S_1": [36.0, 35.99, 35.98, ...],
  "S_2": [34.0, 34.01, 34.02, ...],
  "q": [3.0e-9, 3.01e-9, ...],
  "q_sv": [21.0, 21.07, ...],
  "steady_state_reached": true,
  "final_state": [36.18, 33.82],
  "metadata": {
    "t_max_years": 3000,
    "noise_amplitude": 0.0
  }
}
```

### Field Reference

| Field | Type | Description |
|-------|------|-------------|
| `time` | array of float | Time points (years) — length depends on `t_max/dt` and whether the steady-state event terminated early |
| `S_1` | array of float | Box 1 salinity at each time point (psu) |
| `S_2` | array of float | Box 2 salinity at each time point (psu) |
| `q` | array of float | Circulation strength at each time point (1/s) |
| `q_sv` | array of float | Same as `q` but converted to Sverdrups: `q × V_BOX / 1e6` where `V_BOX = 7e15 m³` |
| `steady_state_reached` | bool | True if the integrator stopped early because `max\|dS/dt\| < 1e-10 psu/yr`. Always `false` for stochastic runs (no event detection). |
| `final_state` | [float, float] | `[S_1_final, S_2_final]` |
| `metadata` | object | Echoes `t_max_years` and `noise_amplitude` |

> The response intentionally does not echo the request parameters (your client already knows them) and does not include a `computation_time_sec` field.

### Conversion notes

- `q_sv = q × 7×10⁹` approximately (with `V_BOX = 7 × 10¹⁵ m³` calibrated so default parameters give ~15 Sv)
- The `time` array length is *at most* `floor(t_max / dt) + 1` for deterministic runs, but may be shorter if the steady-state event terminates early. Stochastic runs (`noise_amplitude > 0`) always return `int(t_max/dt) + 1` points.

### Error Response (422)

```json
{
  "detail": [
    {
      "type": "value_error",
      "loc": ["body"],
      "msg": "Value error, T_1 must be greater than T_2"
    }
  ]
}
```

## Examples

### Minimal request (use all defaults)

```bash
curl -X POST http://localhost:8000/api/simulate \
  -H "Content-Type: application/json" \
  -d '{}'
```

### Custom parameters

```bash
curl -X POST http://localhost:8000/api/simulate \
  -H "Content-Type: application/json" \
  -d '{
    "T_1": 25,
    "T_2": 12,
    "S_1_eq": 36,
    "S_2_eq": 34,
    "S_1_init": 36,
    "S_2_init": 34,
    "alpha": 2.0e-4,
    "beta": 1.0e-3,
    "k": 3.0e-9,
    "lam": 3.0e-11,
    "t_max": 5000,
    "dt": 2.0,
    "noise_amplitude": 1e-5
  }'
```

### Python client

```python
import requests
import matplotlib.pyplot as plt

params = {
    "T_1": 25, "T_2": 5,
    "S_1_eq": 36, "S_2_eq": 34,
    "S_1_init": 36, "S_2_init": 34,
    "alpha": 2.0e-4, "beta": 1.0e-3,
    "k": 3.0e-9, "lam": 3.0e-11,
    "t_max": 3000, "dt": 1.0,
    "noise_amplitude": 0.0,
}

r = requests.post('http://localhost:8000/api/simulate', json=params)
data = r.json()

print(f"Final AMOC strength: {data['q_sv'][-1]:.1f} Sv")
print(f"Steady state: {data['steady_state_reached']}")

plt.plot(data['time'], data['q_sv'])
plt.xlabel('Time (years)')
plt.ylabel('AMOC (Sv)')
plt.show()
```

### JavaScript / fetch

```javascript
const params = {
  T_1: 25, T_2: 5,
  S_1_eq: 36, S_2_eq: 34,
  S_1_init: 36, S_2_init: 34,
  alpha: 2e-4, beta: 1e-3,
  k: 3e-9, lam: 3e-11,
  t_max: 3000, dt: 1, noise_amplitude: 0
};

const r = await fetch('http://localhost:8000/api/simulate', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify(params)
});
const data = await r.json();
console.log(`Final AMOC: ${data.q_sv.at(-1).toFixed(1)} Sv`);
```

## Common Use Cases

### 1. Sweep T₂ to find the tipping point manually

```python
for T_2 in [5, 8, 10, 12, 14, 16, 18]:
    r = requests.post('http://localhost:8000/api/simulate',
                      json={"T_2": T_2})
    q_final = r.json()['q_sv'][-1]
    print(f"T_2 = {T_2}°C (ΔT = {25 - T_2}°C) → q = {q_final:.2f} Sv")
```

### 2. Stochastic ensemble at a single parameter set

```python
import numpy as np
n_runs = 20
finals = []
for _ in range(n_runs):
    r = requests.post('http://localhost:8000/api/simulate', json={
        "T_2": 12, "noise_amplitude": 3e-5, "t_max": 3000
    })
    finals.append(r.json()['q_sv'][-1])

print(f"Mean: {np.mean(finals):.2f} Sv, Std: {np.std(finals):.2f} Sv")
print(f"Collapse fraction: {sum(1 for q in finals if q < 5) / n_runs:.0%}")
```

### 3. Long paleoclimate-style run

```python
r = requests.post('http://localhost:8000/api/simulate', json={
    "t_max": 20000, "dt": 10,
    "T_2": 11, "noise_amplitude": 1e-5
})
data = r.json()
import numpy as np
window = 100
rolling_var = [np.var(data['q_sv'][i:i+window])
               for i in range(0, len(data['q_sv']) - window, window)]
print(f"Variance grew? {rolling_var[-1] > rolling_var[0]}")
```

## Notes on Stochastic Mode

When `noise_amplitude > 0`:

- The solver switches from `scipy.integrate.solve_ivp` (RK45, deterministic, with steady-state event detection) to a fixed-step **Euler-Maruyama** integrator
- Each call uses a **fresh, unseeded** random draw — re-running with identical parameters produces a different trajectory each time
- `steady_state_reached` is always `false` for stochastic runs (no event detection)
- The output has exactly `int(t_max/dt) + 1` points

To compare deterministic vs. stochastic, run twice:

```python
det = requests.post(url, json={**params, "noise_amplitude": 0}).json()
sto = requests.post(url, json={**params, "noise_amplitude": 1e-5}).json()
```

---

## Performance Tips

1. Large `t_max` with very small `dt` makes stochastic runs slow (linear in number of Euler steps). Deterministic RK45 uses adaptive stepping internally and ignores `dt` for accuracy — `dt` only controls the output grid.
2. To sweep many parameter sets, batch the calls with `concurrent.futures` or `httpx.AsyncClient`. The backend serves up to 2 simulations in parallel via its `ThreadPoolExecutor`.
3. For full S-curve studies, use [`/api/bifurcation`](/api-reference/bifurcation) instead of looping `/api/simulate`.

## Error Cases

### Invalid JSON

```bash
curl -X POST http://localhost:8000/api/simulate -d 'not json'
```
→ FastAPI returns 422 with a JSON-decode error.

### Constraint violation

```bash
curl -X POST http://localhost:8000/api/simulate \
  -H "Content-Type: application/json" \
  -d '{"T_1": 5, "T_2": 25}'
```

```json
{
  "detail": [{
    "type": "value_error",
    "loc": ["body"],
    "msg": "Value error, T_1 must be greater than T_2"
  }]
}
```

### Out-of-bounds parameter

```bash
curl -X POST http://localhost:8000/api/simulate \
  -H "Content-Type: application/json" \
  -d '{"alpha": 1.0}'
```

```json
{
  "detail": [{
    "type": "less_than_equal",
    "loc": ["body", "alpha"],
    "msg": "Input should be less than or equal to 0.0003"
  }]
}
```
