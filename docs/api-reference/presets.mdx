# GET /api/presets

Return all preset parameter configurations as a JSON object. Presets are curated parameter sets representing different AMOC regimes (strong, weakened, collapsed, Stommel original).

---

## Endpoint

```
GET /api/presets
```

No query parameters or request body.

---

## Request

```bash
curl http://localhost:8000/api/presets
```

---

### Field Reference

| Field | Type | Description |
|-------|------|-------------|
| `presets` | array | List of preset objects |
| `presets[].name` | string | Programmatic snake_case identifier |
| `presets[].description` | string | Human-readable summary |
| `presets[].params` | object | Complete parameter set ready to POST to `/api/simulate` |

> The list is ordered: `normal_amoc`, `weakened_amoc`, `collapsed_amoc`, `stommel_original`.

The four presets are identical except for `T_2` (and, for Collapsed, `S_2_init`):

| Preset | `T_2` | `S_2_init` | ΔT |
|--------|------:|-----------:|---:|
| `normal_amoc` | 5.0 | 34.0 | 20 °C |
| `weakened_amoc` | 12.0 | 34.0 | 13 °C |
| `collapsed_amoc` | 20.0 | 36.5 | 5 °C |
| `stommel_original` | 7.5 | 34.0 | 17.5 °C |

See the [Presets](/simulator/presets) page for what each represents physically.

## Examples

### List preset names from the shell

```bash
curl -s http://localhost:8000/api/presets | jq '.presets[].name'
# "normal_amoc"
# "weakened_amoc"
# "collapsed_amoc"
# "stommel_original"
```

### Python — load a preset and run it

```python
import requests

# Get all presets
resp = requests.get('http://localhost:8000/api/presets').json()
presets = resp['presets']

# Find Normal AMOC by name
normal = next(p for p in presets if p['name'] == 'normal_amoc')

# Run the simulation with its params
sim = requests.post('http://localhost:8000/api/simulate',
                    json=normal['params']).json()

print(f"Preset: {normal['name']}")
print(f"  description: {normal['description']}")
print(f"  final AMOC:  {sim['q_sv'][-1]:.1f} Sv")
print(f"  steady state: {sim['steady_state_reached']}")
```

### Python — compare all presets at once

```python
import requests

presets = requests.get('http://localhost:8000/api/presets').json()['presets']

print(f"{'Preset':<20} {'T_2':>6} {'ΔT':>6} {'q_sv_final':>12}  steady?")
print("-" * 55)
for p in presets:
    sim = requests.post('http://localhost:8000/api/simulate',
                        json=p['params']).json()
    T_2 = p['params']['T_2']
    dT  = p['params']['T_1'] - T_2
    q_f = sim['q_sv'][-1]
    print(f"{p['name']:<20} {T_2:>6.1f} {dT:>6.1f} {q_f:>10.2f} Sv  {sim['steady_state_reached']}")
```

### JavaScript — populate a UI

```javascript
const { presets } = await fetch('http://localhost:8000/api/presets').then(r => r.json());

const container = document.getElementById('preset-buttons');
presets.forEach(preset => {
  const btn = document.createElement('button');
  btn.textContent = preset.name.replace('_', ' ');
  btn.title = preset.description;
  btn.onclick = () => loadPreset(preset.params);
  container.appendChild(btn);
});
```

### Bash — loop over presets

```bash
#!/bin/bash
presets=$(curl -s http://localhost:8000/api/presets)

for name in normal_amoc weakened_amoc collapsed_amoc stommel_original; do
  params=$(echo "$presets" | jq -c ".presets[] | select(.name==\"$name\") | .params")
  q_final=$(curl -s -X POST http://localhost:8000/api/simulate \
              -H "Content-Type: application/json" \
              -d "$params" | jq '.q_sv[-1]')
  echo "$name -> q = $q_final Sv"
done
```

## Use Cases

### 1. Frontend initialisation

When the simulator loads, fetch presets to populate the preset buttons (the React frontend already does this in `useSimulation.ts`).

### 2. Reproducible benchmarks

Run all four presets and assert their final `q_sv` values match expected ranges — useful as an integration smoke test.

### 3. Quick sweeps from a known starting point

```python
preset = requests.get('http://localhost:8000/api/presets').json()['presets'][0]
params = preset['params'].copy()

# Sweep T_2 starting from Normal AMOC
for T_2 in range(5, 21):
    params['T_2'] = T_2
    q_final = requests.post('http://localhost:8000/api/simulate',
                            json=params).json()['q_sv'][-1]
    print(f"T_2={T_2:>4}°C → q={q_final:6.2f} Sv")
```

## FAQ

### Q: Can I add my own preset?

A: Not via the API — presets are hard-coded in `backend/api.py` (the `get_presets` handler). Edit that file (and the corresponding `loadPreset` button in the frontend's `ParameterControls.tsx`) to add new ones.

### Q: Why are the names in `snake_case` instead of human-readable?

A: They're stable identifiers used by the frontend's `loadPreset(name)` call. The human-readable label is in the `description` field.

### Q: How do I get the bifurcation diagram for a preset?

A: Take the preset's `params`, drop the integration-only fields (`S_1_init`, `S_2_init`, `t_max`, `dt`, `noise_amplitude`), and POST the rest to `/api/bifurcation`:

```python
preset = requests.get('http://localhost:8000/api/presets').json()['presets'][0]
bif_params = {k: v for k, v in preset['params'].items()
              if k in {'T_1', 'T_2', 'S_1_eq', 'S_2_eq',
                       'alpha', 'beta', 'k', 'lam'}}
bif = requests.post('http://localhost:8000/api/bifurcation',
                    json=bif_params).json()
```

## Error Handling

This endpoint is read-only and rarely fails. If the backend is up, the response is always 200 with the static preset list. If you get a connection error, check that the backend is running:

```bash
curl http://localhost:8000/api/health
```
